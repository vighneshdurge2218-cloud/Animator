## Part 3: Image Generation Pipeline (Visual Prompt Engineering)

### 1. Goal
Translate text descriptions of scenes into gorgeous, professional-quality visual assets.

### 2. High-Quality Prompts Enhancement
Standard user-generated visual prompts are often too simple. To guarantee a high-end, premium aesthetic, ANIMaiTOR applies a master style template before sending the prompt to the generator:

> **Enhancement Formula:**
> `[Script Style] style, high-end 3D animation film still, Pixar Disney style, Unreal Engine 5 render. [Script Title]: [Visual Prompt]. Featuring a consistent main character protagonist in the shot. Masterpiece, highly detailed, vibrant colors, dramatic cinematic lighting, 8k resolution.`

### 3. Execution Pipeline
* **API Service:** **Pollinations.ai** (Free, no-auth GPU cluster).
* **Generator Strategy:**
  - **Attempt 1 (Flux Model):** Premium, detailed image generator, outstanding at interpreting complex spatial instructions and rendering textures.
  - **Attempt 2 (Turbo Model Fallback):** Extremely fast generation model utilized if the Flux queue is congested or returns a rate limit (`429`).
* **Error Handling:** If an image fails to load entirely, the backend uses a placeholder to ensure the video rendering pipeline doesn't crash.

### 2. Background Music & Sidechain Compression
* **Music Source:** Premium background tracks downloaded dynamically from **Pixabay** (e.g., Epic Cinematic, Chill Lo-Fi, Documentary).
* **Sidechain Compression Algorithm:**
  When mixing voice narration and music, simple volume reduction makes the audio sound cheap. ANIMaiTOR implements **Sidechain Compression** via FFmpeg:
  ```
  [vo2][duckedbgm]amix=inputs=2:duration=first
  ```
  - The voiceover track acts as a trigger.
  - Whenever there is narration audio, the background music volume is instantly compressed (ducked) to **12%**.
  - During pauses in speech, the music swells back up seamlessly, providing a professional broadcast finish.

## Part 5: Video Animation (SVD-XT Deep Dive & Fallbacks)

### 1. Primary Engine: Stable Video Diffusion (SVD-XT)
* **What it is:** SVD-XT is a latent diffusion model optimized for generating video clips from a static conditioning image. It predicts structural and temporal frame sequences using a noise predictor trained on motion trajectories.
* **Hosting Model:** Run on an external **Google Colab GPU Server (T4 GPU)** bridged securely via Cloudflare.

#### 🛠️ Memory-Safe Optimization Algorithms
T4 GPUs on the free tier of Colab suffer from strict VRAM constraints ($15\text{ GB}$) and system memory limitations ($12.7\text{ GB}$). A raw SVD-XT pipeline instantly crashes. ANIMaiTOR solves this with complex optimizations:

1. **Sequential CPU Offloading (`enable_sequential_cpu_offload`)**
   - *Problem:* Loading the entire UNet model into memory consumes over $8\text{ GB}$ of RAM, causing immediate system crashes.
   - *Solution:* Pages each layer of the model from system RAM to GPU VRAM *only* when it is actively executing, lowering system memory consumption to $\sim 1.5\text{ GB}$.
2. **Attention Slicing (`enable_attention_slicing(1)`)**
   - Computes multi-head attention sequentially (one slice/head at a time) rather than in parallel, vastly reducing peak GPU memory usage.
3. **Decoupled Parameters**
   - **Resolution:** Resized to $512 \times 288$ (reduces VRAM consumption by $21\%$ compared to $576 \times 320$).
   - **Frame Count:** 14 frames (fits within memory constraints).
   - **Inference steps:** 20 denoising steps (speeding up generation by $20\%$ with no visual degradation).
   - **Decode Chunk Size:** 4 frames at a time.

### 2. Fallback: Cinematic Ken Burns Effect
If the GPU server is offline, the backend executes an automatic mathematical simulation:
* **Zoom & Pan Filters:** FFmpeg's `zoompan` generates dynamic, randomized directional movement:
  ```
  zoompan=zoom='min(zoom+0.002,1.12)':x='iw/2-(iw/zoom/2)+20':y='ih/2-(ih/zoom/2)+10'
  ```
* **Filmic Grade:** Applies a color-grade filter:
  - Contrast boosted by $6\%$.
  - S-Curve applied using RGB curve overlays (`curves=all='0/0 0.1/0.08 0.5/0.5 0.9/0.92 1/1'`).
  - Filmic vignette + fine grain overlays (`vignette=PI/5`, `noise=alls=3:allf=t+u`).

## Part 6: FFmpeg Assembly, Transitions & Post-Production

### 1. Timeline Synchronization
* **SVD output:** A 2.0-second video clip.
* **Narration duration:** Anywhere from 3 to 8 seconds.
* **Algorithm:** The video is slowed down or stretched using the `setpts` filter to match the voiceover length exactly:
  ```
  setpts=(scene_duration / SVD_clip_duration) * PTS
  ```

### 2. Transition Blend Chain (xfade)
To avoid jarring cuts, scenes are blended using **xfade** transitions (e.g., `fade`, `wipeleft`, `circlecrop`, `dissolve`) with a $0.5\text{-second}$ overlap. 
The offset is accumulated programmatically:
```
Offset_Scene_N = Duration_Scene_1 + ... + Duration_Scene_N-1 - (N-1 * Transition_Duration)
```

### 3. Rendering Pipeline
Once all filters are mapped, FFmpeg encodes the file:
- **Video Encoder:** `libx264` (Preset: `slow` or `ultrafast` fallback, Constant Rate Factor: `crf 18` for pristine visual fidelity).
- **Audio Encoder:** `aac` (Bitrate: `192k`, Sample Rate: `44100Hz`).
- **FastStart Optimization:** `-movflags +faststart` shifts metadata (moov atom) to the beginning of the file, allowing instant, lag-free video streaming in web browsers.
