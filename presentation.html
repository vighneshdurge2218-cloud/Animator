ANIMaiTOR Deep Dive

Under-the-hood explanation of the models and mathematical algorithms powering our automated AI Movie Studio.

🤖 Pure AI Pipeline

Every single segment (Scripts, Stills, Voices, Motion, Editing) runs using separate specialized state-of-the-art architectures.

Model 1: Scripting

The Scriptwriter Model

Algorithm: Autoregressive Token Prediction (Transformer Architecture)

How it operates under the hood:

It reads your text input and calculates probabilities to predict the next word (token) one-by-one. It uses a mathematical Self-Attention mechanism, looking back at all prior words to ensure logical story flow and scene sequencing.

The system forces the model to structure the visual scenery descriptions and narrated voices strictly inside a JSON payload.

🔑 Active Models:

Qwen-2.5-72B (Creative), Gemini-2.5-Flash (Fast), LLaMA-3.3-70B (Fallback).

Model 2: Scenic Stills

The Scenery Photographer

Algorithm: Flow Matching & Latent Diffusion

How it operates under the hood:

The model starts with a grid of completely random mathematical noise (television static). Over 20-30 denoising steps, the algorithm guides and subtracts this static according to the scene prompt.

It encodes text descriptions into a latent mathematical space, shifting the noise step-by-step until a sharp 3D visual render is formed.

🔑 Style Optimization:

Applies an automated Prompt Wrapping filter (forcing uniform 3D Pixar styles and consistent character presence).

Model 3: Sound Department

The Voice Actor Model

Algorithm: Neural Vocoder Waveform Synthesis

How it operates under the hood:

It translates text sentences into acoustic phonetic syllable structures. A deep neural vocoder predicts speech pacing, volume waves, and intonations to output highly realistic narration waves at **44,100Hz**.

It features realistic human details like natural intakes of breath and contextual inflections.

🔑 Soundboard Mixing:

Uses a custom **FFmpeg Sidechain Compressor** algorithm.

Model 4: Video Motion

The Motion Animator

Algorithm: Temporal Latent Diffusion (3D UNet)

How it operates under the hood:

Takes the 2D still image and injects noise. SVD-XT uses **temporal layers** trained on physics, gravity, and motion to predict how the pixels must flow across 14 consecutive frames, generating a 2.0-second video clip.

To fit on free Google Colab T4 GPU hardware, we engineered critical memory offloads.

🔑 Memory Offloading:

Sequential CPU Offload: Swaps AI layers to GPU one-by-one, cutting RAM overhead from 8GB to 1.5GB. Also uses attention slicing.

Model 5: Video Compile

The Editor Engine (FFmpeg)

Algorithm: Timeline Interpolation & xfade Blends

How it operates under the hood:

Fuses variables together. First, it uses the `setpts` timing filter to slow down the 2-second visual clips to match the exact duration of the spoken voiceover files.

Second, it overlaps adjacent scene boundaries by $0.5\text{ seconds}$ and runs a pixel-by-pixel cross-fade (`xfade`) transition matrix.

🔑 Output Encoding:

Encodes using `libx264 (CRF 18)` and pushes the file header atom to the front (`+faststart`).
